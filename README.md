# Система обнаружения мошеннических отзывов

Рабочий MVP по теме ВКР: backend на FastAPI, обучение модели классификации отзывов, хранение данных в SQLite, веб-интерфейс для загрузки данных и проверки отзывов.

## Возможности
- импорт отзывов из CSV / JSON / JSONL / TXT(JSONL);
- автоматическое приведение полей к единой схеме;
- поддержка полей из диплома: `nmId`, `productValuation`, `color`, `text`, `answer`, `fake_label`;
- обучение модели бинарной классификации;
- вычисление метрик качества;
- сохранение результатов анализа в SQLite;
- веб-форма для предсказания по одному отзыву;
- REST API для интеграции.

## Структура проекта
- `app/main.py` — FastAPI приложение;
- `app/ml.py` — загрузка данных, предобработка, обучение и предсказание;
- `app/db.py` — работа с SQLite;
- `app/schemas.py` — Pydantic-схемы;
- `app/templates/` — HTML-шаблоны;
- `data/sample_reviews.json` — демонстрационные данные;
- `models/` — сохраненные модель и метаданные.

## Быстрый запуск
```bash
cd Trustberry
python -m uvicorn app.main:app --reload
```

После запуска:
- интерфейс: `http://127.0.0.1:8000/`
- Swagger UI: `http://127.0.0.1:8000/docs`

## Форматы входных данных
Поддерживаются поля:
- `nmId` или `product_id`
- `productValuation` или `rating`
- `color`
- `text` или `review_text`
- `answer` или `seller_answer`
- `fake_label` или `label`

Файл `chastdannih.txt` из материалов диплома можно загружать напрямую: приложение обрабатывает его как JSONL/TXT с одной JSON-записью на строку.

## Хранение базы данных
По умолчанию система пытается использовать `data/reviews.db`. Если проект находится на диске, где SQLite не может создать журнал, приложение автоматически переключается на локальный путь `C:\Temp\trustberry\reviews.db`.

## Замечание
Если в реальном датасете нет `fake_label`, можно:
1. добавить ручную разметку;
2. использовать полуавтоматическую слабую разметку как стартовый этап;
3. затем переобучить модель на реальной размеченной выборке.
